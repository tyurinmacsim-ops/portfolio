# Руководство по эксплуатации Vault (шаблон)

## 1. Предусловия

- Vault развернут в Kubernetes в режиме HA с backend `raft`.
- Настроен auto-unseal через Yandex KMS.
- Настроен TLS для Vault listener и ingress.
- Подготовлен bucket и учетные данные для snapshot backup (опционально, но рекомендуется).

## 2. Одноразовый bootstrap

1. Выполнить `vault operator init` на первом pod.
2. Сохранить unseal keys и initial root token в утвержденном защищенном хранилище.
3. Проверить состояние кластера Raft (leader и peers).
4. Включить Kubernetes auth и создать роли/политики по принципу наименьших привилегий.

## 3. Регулярная эксплуатация (ежедневно/еженедельно)

- Проверка leader и peers: `vault operator raft list-peers`.
- Проверка seal/unseal статуса: `vault status`.
- Проверка выполнения snapshot job и загрузки backup в Object Storage.
- Ротация операционных токенов согласно политике безопасности.

## 4. Политика резервного копирования

- Полный Raft snapshot каждые 6 часов.
- Хранение snapshot минимум 14 дней в Object Storage.
- Тестовое восстановление из актуального snapshot не реже 1 раза в месяц.

## 5. Сценарии восстановления

### 5.1 Потеря одного pod

- Kubernetes должен автоматически пересоздать pod.
- Проверить, что pod присоединился к кластеру и автоматически расшифрован через KMS.

### 5.2 Потеря нескольких pod при сохраненном quorum

- Восстановить недоступные pod.
- Проверить лидерство и консистентность Raft-кластера.

### 5.3 Потеря quorum / полное восстановление

1. Поднять чистые Vault pod той же мажорной версии.
2. Запустить процедуру восстановления на целевом кластере.
3. Восстановить данные из последнего валидного snapshot:
   `vault operator raft snapshot restore <file>`.
4. Проверить статус unseal (при auto-unseal должен выполниться автоматически).
5. Провалидировать auth mounts, политики и доступ к секретам приложений.

## 6. Контроли безопасности

- Не хранить reviewer JWT, root token и долгоживущие plaintext-секреты в Git.
- Ограничить доступ к backup bucket только выделенному service account.
- Обязать MFA/ограниченные роли для операторских действий.
- Включить и регулярно проверять аудит-логи Vault.

## 7. Артефакты для аудита/комплаенса

- Манифесты и версия Vault (HA + Raft конфигурация).
- Идентификатор KMS key и IAM bindings.
- Расписание backup и отчеты по тестам восстановления (дата, оператор, результат).
- Манифесты интеграции с Kubernetes (auth, роли, способ доставки секретов).
